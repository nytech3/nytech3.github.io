操作系统_操作系统中常见的页面置换算法

发表于 2021-05-08 分类于 技术 阅读次数：

文章推荐: 页面置换算法

常见的页面置换算法有以下几种:

OPT页面置换算法(最佳页面置换算法)

基本思路

当一个缺页中断发生时，对于保存在内存中的每一个逻辑页面，计算在它在下一次的访问之间的等待时间，从中选择等待时间最长的那个，作为被置换的页面。
这种思路是最理想的情况，但事实上，一个操作系统并无法知道每一个页面下一次被访问的时机。虽然这种算法无法被实现，但是可以作为一种度量，如果有一种算法其效率最接近OPT，那么这种算法无疑是优秀的算法。

算法举例

已知有0，1，2，3，4，5，6，7，8，9十个页面，其使用顺序为9->0->2->2->3->1->7->4->7->9->1，且系统为进程仅分配了3个页面大小的物理内存，这时其物理内存的置换情况如下:

  1. 9<- : 9 <-
  2. 0<- : 9 <- 0 <-
  3. 2<- : 9 <- 0 <- 2 <-
  4. 2<- : 9 <- 0 <- 2 <-
  5. 3<- : 9 <- 3 <- 2 <-
  6. 1<- : 9 <- 1 <- 2 <-
  7. 7<- : 9 <- 1 <- 7 <-
  8. 4<- : 9 <- 4 <- 7 <-
  9. 7<- : 9 <- 4 <- 7 <-
  10. 9<- : 9 <- 4 <- 7 <-
  11. 1<- : 1 <- 4 <- 7 <-

先进先出算法（first-in first-out ，FIFO ）

将最早装载至内存中进入队列的页面置换

缺点:

  1. 会将经常使用的页面置换出来
  2. 进程分配物理页面数增加时 缺页不一定减少(Belady 现象)

Belady 现象

采用 FIFO 等算法时 可能出现分配的物理页面数增加 缺页次数反而升高的异常现象

产生 Belady 现象的原因

FIFO 算法的置换特征 与 进程访问内存的动态特征矛盾
被它置换出去的页面 并不一定是 进程 近期不会访问的页面

最近最久未使用算法（least recently used， LRU ）

LRU认为某些页面长时间未被访问 则它们在将来还可能会长时间不会访问，故其选择最长时间没有被引用的页面进行置换。缺页时LRU计算内存中每个逻辑页面的上一次访问时间，选择上一次使用到当前时间最长的页面置换出去。

实现:
系统维护一个按最近一次访问时间排序的链表,链表首节点是最近刚刚访问过的页面,链表尾节点是最久未使用的页面,访问内存时找到相应页面并将其移到链首,缺页时置换链表尾节点的页面.

时钟页面置换算法（Clock）

时钟页面置换是一种最近未使用使用算法，其将内存中的页面像环形队列一样串接起来，通过指针轮转各个页（这个过程就像是时钟的指针的转动），通过各个页中的标识符，找到最近未使用的页面进行替换，页面中有两个标识符号,access和modify,access来表示最近被访问，modify用于表示最近被修改，access为1时被认为最近被访问了，modify为1时被认为是最近被修改。

  1. 指针轮转到access位为1的页面时，将该页的access位置为0，表示时钟在轮转过所有页面之后该页最近仍未被使用过。
  2. 若指针再轮转一周之后，该页仍未被使用，则再检查modify是否为0，若为1,则将modify置为0，指针继续轮转，不进行页面置换，这里的目的是当页面被修改时需要将页面写回磁盘，频繁的写操作需要耗费大量资源。当modify为0时，则将页面置换。
  3. 若使用的页面已在内存中存在，则将该页面的access位置为1，若该页面被修改则将modify同时置为1，同时将该页面移动至指针之后。

最不常用算法（least frequently used，LFU）

每个页面都有一个计数器，访问页面时，计数器+1，缺页时置换访问次数最少的页

缺点:

  1. 一开始页面频繁使用，导致计数器数值较大，导致不再使用的页面一直无法置换出去。（解决方法：数值较大的计数器，定期减少其数值）

LRU 和 LFU

最不常用算法和最近最久未使用页面置换算法的区别主要在于关注点不同

  1. 最不常使用算法关注的是使用次数的多少，使用次数越多越能得到驻留的机会。
  2. 最近最久未使用页面置换算法关注的是最近是否有使用，若最近越常使用，得到的驻留机会更大。
