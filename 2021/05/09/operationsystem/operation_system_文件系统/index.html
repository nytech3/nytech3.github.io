文件系统

用户组

作用

  1. 可以设定文件只能被某些用户和用户组访问，进行权限控制。
  2. 一个用户可以被设定多个用户组，这样用户就可以拥有自身所有用户组的文件访问权限。

Linux文件系统简介

在linux中，所有被操作系统管理的资源，例如网络接口卡，磁盘驱动器，打印机，输入输出设备，普通文件和目录都可以看作是一个文件。也就是说在linux中有一个重要的概念:一切都是文件。

inode介绍

inode是linux/unix文件系统的基础。

文件存储在硬盘上，硬盘最小的存储单位为扇区(Sector),每个扇区存储512个字节。

操作系统读取文件的时候不会一个扇区一个扇区的读，而是一个块(block)一个块的读,一个块由多个扇区组成，一个块大小一般为4kb，即8个扇区。

一个文件可能会占有多个Block，但是一个块只能存放一个文件。一个文件大小大于一个块的大小时，该文件则会被存储在多个块中，但是一个块只能存储一个文件，如果有两个文件存储在同一个块中时，inode则无法准确的定位到文件。

文件都存储在块中，但是文件还拥有元数据消息，那就需要找到存储元数据（文件创建者，文件创建时间，文件大小等等）的地方，这种存放元数据的地方就叫inode（index node），即索引节点。

每一个文件都有一个inode存储元信息。

可以使用stat命令可以查看文件的inode信息,每个inode都有自己的一个编号，Linux/Unix操作系统不使用文件名来区分文件，而是使用inode的编号来区分不同文件。

  1. dev_t st_dev; //device 文件的设备编号
  2. ino_t st_ino; //inode 文件的i-node
  3. mode_t st_mode; //protection 文件的类型和存取的权限
  4. nlink_t st_nlink; //number of hard links 连到该文件的硬连接数目, 刚建立的文件值为1.
  5. uid_t st_uid; //user ID of owner 文件所有者的用户识别码
  6. gid_t st_gid; //group ID of owner 文件所有者的组识别码
  7. dev_t st_rdev; //device type 若此文件为装置设备文件, 则为其设备编号
  8. off_t st_size; //total size, in bytes 文件大小, 以字节计算
  9. unsigned long st_blksize; //blocksize for filesystem I/O 文件系统的I/O 缓冲区大小.
  10. unsigned long st_blocks; //number of blocks allocated 占用文件区块的个数, 每一区块大小为512 个字节.
  11. time_t st_atime; //time of lastaccess 文件最近一次被存取或被执行的时间, 一般只有在用mknod、utime、read、write 与tructate 时改变.
  12. time_t st_mtime; //time of last modification 文件最后一次被修改的时间, 一般只有在用mknod、utime 和write 时才会改变
  13. time_t st_ctime; //time of last change i-node 最近一次被更改的时间, 此参数会在文件所有者、组、权限被更改时更新

对于Ext2系统来说，inode还记录了文件实际存储在硬件上的block号码，inode要记录的东西很多，但其大小只能有128kb，一个block号码就占了4byte，对于一个较大的文件，inode根本无法存储如此之多的信息，为此我们将block号码区分成15个区块，12个直接，1个间接，1个双间接，1个三间接记录区。

其中12个直接记录块可以直接指向block号码，间接就是再拿block块来记录指向block号码的block记录区，如果文件太大，就会使用间接的block来记录，三间接记录区同理于双间接记录区。

这样子，当单个block为1kb时，inode就可以指向

  1. 直接记录区: 12*1k=12k
  2. 间接接可以指向(1024b/4b)*1k=256k,每笔block号码记录会耗费4bytes，因此1k能存储256笔记录，所以一个间接记录区存储的文件大小为256kb。
  3. 双间接为256*256*1k
  4. 三间接为256*256*256*1k

总大小为12+256+256*256+256*256*256=16GB
因此当单个block为1kb大小时，能容纳的单个文件最大为16GB，系统总容量为2TB。当然，当单个block块更大时，所能记录的文件大小也会变大。

目前Ext4文件系统已经支持的inode容量为256bytes，更大的inode容量可以记录更大的文件，单一文件大小达到了了16KB且单一文件系统总容量达到了1EB。

Linux文件类型

使用ls能够查看文件的文件类型，如返回结果中的drwxr-xr-x的第一位d代表的是directory，即目录文件。
常见的文件类型主要如下几种:

  1. - : regular file
  2. d : directory
  3. c : character device file
  4. b : block device file
  5. s : local socket file
  6. p : named pipe
  7. l : symbolic link

Regular file

普通文件类型是linux系统中最常见的文件类型，它包含了很多不同的文件，例如文本文件，图片，二进制文件，shared libraries, etc.。普通文件可以使用touch命令创建。

Directory

目录文件是linux中第二常见的文件类型，用于表示和管理系统中的文件，目录文件包含了文件名和子目录名。，打开目录实际上就是打开目录文件，可以使用mkdir命令创建。

Block file type

设备文件，这些文件一般都在/dev中，这种文件可以使用fdisk创建，设备文件和字符文件可用于跟外围设备做交互，如硬盘，软盘。

Character device

字符设备用来访问字符设备比如硬盘。

sockets

套接字可用于进程之间的通信

Named Pipes

可用于进程之间的信息通信，可以使用mknod命令创建。

Symbolic Links

符号链接文件，将文件链接到另外的文件，其在linux中可分为软链接和硬链接，硬连接的inode跟其父文件相同，

linux常见目录

推荐阅读 :Filesystem Hierarchy Standard

/ ROOT(根目录)

  1. 操作系统下的所有文件都是挂在根目录下的
  2. 只有root用户才有这个目录下的写权限

/root – Root Home Directory

root用户的home文件夹

/bin User Binaries(用户二进制文件)

  1. 包含可执行的二进制文件
  2. 单用户模式下常见的linux命令就放在这个目录下
  3. 系统下被所有用户使用的命令就放在这个目录下
  4. 例如: ps,ls,ping,grep,cp

/sbin System Administration Binaries(系统管理员二进制文件)

  1. 与/bin相似，/sbin中包含了可执行的二进制文件
  2. 这个目录下存储的可执行二进制文件一般是超级管理员使用的命令。
  3. 例如:reboot，iptables,fdisk,ifconfig

/boot – Static Boot Files

存放用于引导系统的各种重要文件，引导的加载文件放在此目录下，但是引导相关的配置文件fangzai/etc下。

/dev – Device Files

linux下所有设备都可以用文件表示，此目录下包含了所有的linux的外部设备，但这个文件下并非所有文件都代表了硬件。/dev/null代表着空设备，你可以向其输入任何数据，但任何写入它的数据都会被抛弃，通常用于处理不需要的输出流，当然它也可以被当作空的输出流。/dev/zero在类UNIX 操作系统中, /dev/zero 是一个特殊的文件，当你读它的时候，它会提供无限的空字符(NULL, ASCII NUL, 0x00)。一个典型用法是用它提供的字符流来覆盖信息，另一个常见用法是产生一个特定大小的空白文件。

/dev/null用途:

  1. 想要忽略一些执行命令的异常时可使用
  2. 想要在不删除文件只删除一些文件的内容的情况下可使用

/etc Configuration Files(配置文件)

  1. 用来存放系统的主要配置文件。用户的配置文件存在用户的home文件下。
  2. 但是只有root才有权利进行修改。

/home Home Folders

  1. 每个用户都会有自己的一个home文件夹
  2. 每个人只能修改自己home文件夹下的文件，想要修改其他人的文件，需要获取root权限才能修改。

/lib Essential Shared Libraries

/lib 文件夹是 库文件目录 ，包含了对系统有用的库文件。简单来说，它是应用程序、命令或进程正确执行所需要的文件。在 /bin 或 /sbin 目录中的命令的动态库文件正是在此目录中。内核模块同样也在这里。

以 pwd 命令执行为例。执行它需要调用一些库文件。让我们来探索一下 pwd 命令执行时都发生了什么。我们需要使用 strace 命令 找出调用的库文件。

会发现我们使用的 pwd 命令的执行需要调用两个库文件。

/lost+found Recovered Files

每一个linux系统都会有一个/lost+found文件夹，如果系统发生意外崩溃，下一次系统启动时文件系统就会进行校验。那些在系统意外奔溃，未能正确退出的文件就会存在这个文件夹下。

/media Removable Media

该文件夹下主要存放的是可插拔的媒体设备文件，当你插入一张CD时，CD内相关的内容就可以通过这个文件夹下的插入CD后生成的文件夹进行访问。

/mnt Temporary Mount Points

系统管理员安装临时文件系统的安装点，系统提供这个目录是让用户临时挂载其他的文件系统；

/proc Kernel & Process Files

官方解释：

The proc filesystem is a pseudo-filesystem which provides an interface to kernel data structures. It is commonly mounted at /proc. Typically, it is mounted automatically by the system, but it can also be mounted manually using a command such as:

$ mount -t proc proc /proc (it means –> mount -t type device dir, This tells the kernel to attach the filesystem found on device (which is of type type) at the directory dir. )

Most of the files in the proc filesystem are read-only, but some files are writable, allowing kernel variables to be changed.

简单的说， proc 是一个伪文件系统，它提供了内核数据结构的接口。它通常挂载在 /proc 目录下。一般是由系统自动挂载的，不过也可以通过 mount 命令进行手动挂载。proc 文件系统只包含系统运行时的信息（如系统内存、mount 设备信息等），它只存在于内存中而不占用外存空间。它以文件系统的形式，为访问内核数据的操作提供接口。
/proc包含很多目录，其中以进程ID命名的目录代表就是那个进程。这些目录下众多文件包含了进程的信息和统计信息，由内核数据映射而来。

/run – Application State Files

使用df检查/run会发现是一个”tmpfs”(临时文件系统)，因此其不会存储在磁盘上，其只会存储在内存上。里面的东西是系统运行时需要的, 不能随便删除(因此不能存在/tmp中，/tmp中的数据有可能会被删除). 但是重启的时候应该抛弃. 下次系统运行时重新生成。

/srv – Service Data

存放的是服务所需要的一些数据，例如，运行一个website服务时，就会将一些站点的资源文件放在这里。如web、ftp、流媒体等。

/tmp

According to FHS-3.0, /tmp is for temporary files and /run is for run-time variable data. Data in /run must be deleted at next boot, which is not required for /tmp, but still programs must not assume that the data in /tmp will be available at the next program start.

/usr

/usr is the second major section of the filesystem. /usr is shareable, read-only data. That means that /usr should be shareable between various FHS-compliant hosts and must not be written to. Any information that is host-specific or varies with time is stored elsewhere.

Large software packages must not use a direct subdirectory under the /usr hierarchy.

usr是user的缩写，是曾经的HOME目录，然而现在已经被/home取代了，现在usr被称为是Unix System Resource，即Unix系统资源的缩写。

/var – Variable Data Files

用于存放运行时需要改变数据的文件，也是某些大文件的溢出区，比方说各种服务的日志文件（系统启动日志等。）等；

硬件和固件之间有什么不同？

为了使硬件正常运行，很多设备软件由两部分软件组成。加载到实际硬件的代码部分就是固件，用于在固件和内核之间通讯的软件被称为驱动程序。这样一来，内核就可以直接与硬件通讯，并确保硬件完成内核指派的工作。
