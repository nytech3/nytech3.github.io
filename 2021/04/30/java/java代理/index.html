java代理

java代理概念

代理模式是通过使用代理类对象访问真实对象方法的设计模式，这种方式可以在不改变原对象方法的前提下，在代理类方法中增强原有方法能力。常见的场景有日志系统，在原对象方法执行的前后打印日志。

java代理的使用场景

  1. 日志系统
  2. 权限校验
  3. 缓存
  4. 数据同步

java代理模式

静态代理

静态代理即在编码时期，对原对象的属性和方法进行代理以屏蔽对原对象的，这样的代理方式往往延展性较差，举个例子，假设有10个方法需要在执行前后打印日志，就需要写10个代理方法进行调用。

生成的代理类

JDK动态代理在代码运行过程中动态的生成了代理类，在调用代理类的方法时可以在真实的代理方法前后加上代理方法以外的操作，如以上编写的样例，在调用printFruit前后加上了日志打印。

JDK动态代理优点:

  1. JDK动态代理相比静态代理更加灵活，拥有了更高的延展性.
  2. JDK动态代理是JAVA自带的包，无需引入其他依赖.

JDK动态代理缺点:

  1. 只能对接口进行代理.原因是代理动态生成的类继承了Proxy类无法在继承继承其他类。例如上面Fruit的例子Proxy只能implement Fruit接口，而无法extends Apple类。
