源码阅读(六)-Redis中的SADD命令

发表于 2023-09-23 分类于 technology 阅读次数：

工作流程

  1. 查找key是否存在于客户端选择的db中，如果存在则检查对应的类型是否为set，不为set的话直接返回，
    为set类型的话检查新增数据之后是否需要进行编码转化，需要的话则进行转化，如果不存在则将对应的key
    和新创建的set对象值加入db中
  2. 在类型转化时，会判断是否超过对应编码设置的最大大小,只有超过阈值才会发生转化，转化时遍历原来的所有数据，加入新的dict对象中
    a. 当原set编码类型为LISTPACK，并且大小超过服务器设置listpack的最大大小限制时会发生转化，转化的目标编码为OBJ_ENCODING_HT
    b. 当原set编码类型为INTSET，并且大小超过服务器设置intset的最大大小限制时发生转化，转化的目标编码为OBJ_ENCODING_HT
  3. 遍历所有要新增的值，将其加入set对象中，新增成功则将记录下来(added字段+1，用于返回给客户端)，如果之前数据已存在则不+1。
    a. 如果set为OBJ_ENCODING_HT编码，则通过hash计算找到对应的bucket，若数据不存在于对应的bucket中，则将值置入其中
    b. 如果set为OBJ_ENCODING_LISTPACK编码，则通过遍历列表查找对象，不存在则置入队列尾部
    c. 如果set为OBJ_ENCODING_INTSET编码，则通过二分查找找到对应的值，找不到则找到对应的插入点进行插入，如果插入值不为数值类型，则检查是否超过listpack大小阈值，不超过的话转化成LISTPACK，超过的话转化成HT，再进行插入
  4. 如果有新增数据则进行set的sadd事件通知，并将server.dirty加上added，表示有added个数据发生了变更。

    源码解析

    SADD json

    添加一个或者多个对象到set中，如果键不存在则创建键。如果只新增一个对象，其时间复杂度为O(1),如果新增多个对象则时间复杂度为O(n),从2.4.0版本开始SADD命令支持添加多个值，命令结果返回新增的值。
