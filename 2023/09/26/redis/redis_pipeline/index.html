源码阅读(八)-Redis中的MULTI命令(PIPELINE)

发表于 2023-09-26 分类于 technology 阅读次数：

工作流程

  1. 执行MULTI，让客户端进入事务状态（将client中的flags的multi标识位置为1）
  2. 客户端进行命令请求，这时候每个请求都会进行权限校验，确保客户端有权限执行，然后判断是否在执行事务中，
    如果正在执行事务，判断客户端正在监听的key是否已被其他客户端修改，如果没有CLIENT_DIRTY_CAS的标识，
    则将命令放到客户端的事务命令队列中。
  3. 客户端请求EXEC，对所有命令进行权限判断，判断乐观锁，校验通过之后执行所有命令。

源码解析

MULTI & EXEC json

MULTI用于开启一个事务，事件复杂度为O(1)，事务是在1.2.0版本之后引进.

EXEC用于执行所有事务中的事件,时间复杂度依赖于事务中的命令，返回值正常会是一个数值，数组包含了所有命令的执行结果。
