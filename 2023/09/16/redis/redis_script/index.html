EVAL

Redis提供了执行Lua脚本的能力，它允许你使用Lua编程语言来编写脚本并在Redis服务器上执行。
这个功能可以用来实现原子性操作（atomic operations），避免了在客户端和服务器之间进行多次通信。
Lua脚本在Redis中执行是原子性的，这意味着在脚本执行的过程中，不会有其他命令插入，这就避免了一些并发问题。

虽然在执行EVAL时可以只使用key或者arg，但实际上，KEYS和ARGV在Redis Lua脚本中起到的是不同的作用。 KEYS数组用于存储传入脚本的键名，这些键是你在脚本中需要操作的Redis数据结构。而ARGV数组用于存储除键名以外的其他参数。 你可以选择只用ARGV，按照惯例，我们通常将需要操作的键名通过KEYS传入，将其他参数通过ARGV传入。这样做也使得代码更加清晰易读。

EVAL

-> evalCommand :

当服务器收到一个SCRIPT DEBUG命令时，它会开启CLIENT_LUA_DEBUG模式。例如： SCRIPT DEBUG YES, 在这个命令执行后，后续的Lua脚本执行会在调试模式下进行。 这个功能主要是为了Redis的开发和调试使用，它可能会影响Redis的性能，不建议在生产环境中使用。而且，开启CLIENT_LUA_DEBUG模式后，服务器在每次执行Lua脚本时都会打印一些调试信息。
