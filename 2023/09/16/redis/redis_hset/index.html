源码阅读(三)-Redis中的HSET命令

发表于 2023-09-16 分类于 technology 阅读次数：

工作流程

  1. 检查参数：HSET 命令最少需要三个参数，包括哈希表的键、字段和值。如果参数数量不正确，Redis 会返回错误。
  2. 获取哈希表：Redis 会根据给定的键从数据库中获取哈希表对象。如果对象类型不是哈希表，Redis 会返回错误。
  3. 判断字段是否存在：Redis 判断给定的字段是否已经存在于哈希表中。
  4. 设置字段的值：根据字段是否已经存在执行不同的操作：
  5. 如果字段已经存在，Redis 将更新该字段的值，如果新值和旧值不相同，Redis 还会进行额外的内存管理操作，包括释放旧值的内存和分配新值的内存。
  6. 如果字段不存在，Redis 将在哈希表中创建新的字段，并设置其值。
  7. 更新哈希表的元数据：如果添加了新的字段，Redis 还会更新哈希表的元数据，包括字段数量和哈希表的内存占用。
  8. 返回结果：HSET 命令在设置字段的值后会返回一个结果，表示新建了多少个键值对。

源码解析

hset json

该命令为HSET，用于在哈希表中创建或修改字段的值。对于每对添加的字段/值，HSET命令的时间复杂度为O(1)。其组别为”hash”。
从2.0.0开始引入，该命令至少需要4个参数。对应的方法为hsetCommand。在4.0.0版本之后，HSET开始接受多个字段和值作为参数。
