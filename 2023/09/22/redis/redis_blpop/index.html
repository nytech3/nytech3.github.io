源码阅读(五)-Redis中的BLPOP命令

发表于 2023-09-22 分类于 technology 阅读次数：
  1. 检查命令是否有设置超时时间，没有的话直接返回
  2. 遍历所有的键，逐个进行出队列取值，或者阻塞
  3. 检查键是否在client选择的数据库中
  4. 如果对应的键不存在则跳过
  5. 检查对象是否为List，如果不是则返回
  6. 获取键对应的值列表长度，为0则跳过
  7. 判断是否允许阻塞队列，如果不允许则直接返回
  8. 没有数据则阻塞队列

关于阻塞的键是如何通知到客户端的，暂时不分析，大概在分析完所有主要命令之后，会对事件的主流程进行解析（感觉这样到时候看主流程的事件触发会对各个事件比较了解不会磕磕碰碰），
到时候再看看redis对于命令的是否是单线程的，目前还不是很了解。

源码解析

BLPOP json

作用为获取并移除列表中的第一个元素, 如果获取不到则将一直阻塞能获取到,其时间复杂度为O(n),这个n取决于阻塞了多少个key。
可以用Redis的BLPOP实现一个简单的任务队列，生产者将任务作为数据放入一个列表，消费者则使用BLPOP命令从列表中取出任务并
执行。由于BLPOP是一个阻塞命令，所以当队列中没有任务时，消费者会等待直到有新的任务到来。
